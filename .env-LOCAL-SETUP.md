# .env para LOCAL — Guía rápida

## Archivos que hemos creado

1. **`.env`** — Configuración por defecto para LOCAL (ya completa)
2. **`.env.local`** — Overrides opcionales SOLO para tu PC (no se sube)

## ¿Qué usa tu máquina LOCAL?

Cuando ejecutas:
```bash
php -S localhost:8000 -t public_html
```

PHP carga las variables en este orden:
1. `.env.local` (si existe) — sobrescribe todo
2. `.env` (siempre existe) — valores por defecto

## Variables importantes para LOCAL

### Base de datos
```
DB_DRIVER=sqlite
DB_SQLITE_PATH=./db/miatech.sqlite
```
✅ Funciona sin MySQL ni credenciales.

### Almacenamiento
```
STORAGE_DRIVER=local
STORAGE_LOCAL_PATH=./storage
```
✅ Los archivos se guardan en `./storage/` en tu máquina.

### GROQ (IA)
```
GROQ_API_KEY=
```
⚠️ Vacío = funciona pero sin IA.  
Si tienes clave: pégala en `.env.local`:
```
GROQ_API_KEY=gsk_tu_clave_aqui
```

### SMTP (Correo)
```
SMTP_HOST=
SMTP_PASSWORD=
```
⚠️ Vacío = funciona pero sin correos de verdad.

## Primeros pasos

### 1. Crear la BD
```bash
php tools/migrate.php --seed
```
Esto crea el esquema SQLite + datos de prueba.

### 2. Arrancar el servidor
```bash
php -S localhost:8000 -t public_html
```

### 3. Probar
```
GET  http://localhost:8000/api/health
GET  http://localhost:8000/api/config
POST http://localhost:8000/api/auth/login
```

## Si algo no funciona

### 403 en `/api/config`
Causa: colisión de nombres entre la ruta y el archivo `api/config.php`.  
Fix: ver `docs/DUDAS_PENDIENTES.md`.

### HTML en vez de JSON en login
Causa: `storage/tmp/` no escribible → Warning de PHP rompe JSON.  
Fix: asegurate que `storage/tmp/` existe y es escribible.

### Rutas no funcionan
Si los paths relativos (`./db`, `./storage`) no van, usa paths absolutos en `.env.local`:
```
DB_SQLITE_PATH=C:/Users/Andres/proyecto.../db/miatech.sqlite
STORAGE_LOCAL_PATH=C:/Users/Andres/proyecto.../storage
```
